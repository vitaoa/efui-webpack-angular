  1. 简介
    1. SVG是一种可缩放矢量图形（英语：Scalable Vector Graphics，SVG），基于可扩展标记语言（XML），用于描述二维矢量图形的图形格式。
    2. SVG由W3C制定，是一个开放标准。
    3. 简单的理解：它是图形的另一种格式例如它和常见的图片格式.png、.jpg、.gif等是一类。
  2. 嵌入HTML的几种方式
    1. 代码直接嵌入 
      <svg xmlns="http://www.w3.org/2000/svg">
    <line x1="0" y1="0" x2="200" y2="200" style="stroke-width:1;stroke:black"/>
</svg>
    2. img标签src方式引用 
      <img src="svgDemo.svg"/>
    3. a标签href方式引用 
      <a href="svgDemo.svg">转到SVG图像
      该方法只会显示链接文本，只有当用户单击链接，才能跳转至SVG图像。
    4. CSS的background属性引用 
      <div style="width:500;height:500;background:yellow url(svgDemo.svg) repeat"/>
    5. embed 
      <embed src="svgDemo.svg" type="image/svg+xml" />
      默认的大小是 300 x 150
    6. object 
      <object data="svgDemo.svg" type="image/svg+xml" />
      默认的大小是 300 x 150
    7. iframe 
      <iframe src="svgDemo.svg" />
      默认的大小是 304 x 154，因为iframe有边框，边框线宽是 2。

    针对embed、object、iframe：为了确保图像能正常显示，最好手动给svg指定width和height属性。

  3. 基本概念 
    <svg x="0px" y="0px" width="700px" height="200px" viewBox="0 0 550 100" preserveAspectRatio="xMinYMin meet">
     <rect x="10" y="5" fill="white" stroke="black" width="90" height="90"/>
     <circle fill="white" stroke="black" cx="170" cy="50" r="45"/>
     <polygon fill="white" stroke="black" points="279,5 294,35 328,40 303,62 309,94 279,79 258,94 254,62 230,39 263,35"/>
     <line fill="none" stroke="black" x1="410" y1="95" x2="440" y2="6"/>
     <line fill="none" stroke="black" x1="360" y1="6" x2="360" y2="95"/>
</svg>
    1. viewport: 物理窗口

      用 x,y,width,height 这4个属性，在页面上固定的矩形区域。

    2. viewbox: 实物窗口

      定义 SVG 元素在 viewport 中的具体尺寸比例

      注意，在 SVG 中，子标签的所有尺寸都是不能带单位的，因为初始单位就是根据viewport和viewbox确定。

      案例中：viewport 为 [0,0] 到 [700,200]，viewbox 为 [0,0] 到 [550,100]

      SVG 中基本的尺寸则不是 1px，而是 700/550 = 14/11px=1.27px。

      <rect x="10" y="5" fill="white" stroke="black" width="90" height="90"/>
      实际尺寸为 [12.7,6.35] 到 [114.55,114.55]
    3. preserveAspectRatio: 保留横纵比

      用来定义上面 viewport 和 viewbox 相互对齐的方式

      基本格式为：<align> [<meetOrSlice>]
      • align：默认值为 xMidYMid，即为中点重合，viewbox 是通过中心进行延展的。
      • meetOrSlice: 主要就是定义该 SVG 是内嵌，还是裁剪或是 none，即meet || slice || none。
        • meet(默认值): 本意是让 svg 尽可能的显示在 viewport 里，即，会在 rat_x 和 rat_y 中选择最小的值作为缩放标准。
        • slice: 本意是让 svg 完全铺满 viewport，即，会在 rat_x 和 rat_y 中选择最大的值作为缩放标准。

      案例中：viewport 为 [0,0] 到 [700,200]，viewbox 为 [0,0] 到 [550,100]

      rat_x = 700/550 = 1.27；rat_y = 200/100 = 2；

      SVG 中基本的尺寸：当为 meet 的情况，那么实际缩放比例为 1.27；当为 slice 的情况，那么实际缩放比例为 2。

  6. 分组和Path（通常 Path 和 分组通常是一起使用的） 
    <g fill="none" stroke="#000">
    <path d="M7.3 75L25.9 6.8s58.4-6.4 33.5 13-41.1 32.8-11.2 30.8h15.9v5.5s42.6 18.8 0 20.6" />
    <path d="M133.1 58.2s12.7-69.2 24.4-47.5c0 0 4.1 8.6 9.5.9 0 0 5-10 10.4.9 0 0 12.2 32.6 13.6 43 0 0 39.8 5.4 15.8 15.4-13.2 5.5-53.8 13.1-77.4 5.9.1 0-51.9-15.4 3.7-18.6z" />
</g>
    1. Path：
      • path 中 d(data) 属性是用来定义相关线条数据，通常是以 M/m 为起始，代表的就是 move to 的意思
      • 在 path 中，一共可以定义 10 种不同的图形
        • 线型
          • M/m：定义起始点
          • L/l：画线段（Line to）
          • H/h：画水平线
          • V/v：画竖直线
          • Z/z：表示 path 的结束，并且将最后一点和 M/m 标识开头的一点连接起来
          <path d="M10 10 H 90 V 90 H 10 L 10 10"/>
使用 Z
<path d="M10 10 H 90 V 90 H 10 Z" fill="#ccc" stroke="red"/>

          该 path 实际上就是画了一个正方形，宽 = 高 = 80。

        • 曲线：贝塞尔曲线
          • C/c：正统的贝塞尔曲线，需要定义三个点

            基本格式为：C x1 y1, x2 y2, x y (or c dx1 dy1, dx2 dy2, dx dy)

            <path d="M10 10 C 20 20, 40 20, 50 10" stroke="black" fill="transparent"/>
          • S/s：表示一个反射贝塞尔，即在原有贝塞尔上再加一段贝塞尔曲线，一般和 C/c 一起使用

            基本格式为：S x2 y2, x y (or s dx2 dy2, dx dy)

            <path d="M10 80 C 40 10, 65 10, 95 80 S 150 150, 180 80" stroke="black" fill="#ccc"/>
          • Q/q：定义二次贝塞尔曲线，相当于上面传统的贝塞尔来说，更加简单，只需要定义二个点，起始点为 M 定义的点

            基本格式为：Q x1 y1, x y (or q dx1 dy1, dx dy)

            <path d="M10 80 Q 95 10 180 80" stroke="black" fill="#ccc"/>
          • T/t：和 S/s 差不多，也是一个贝塞尔曲线的延长，只需要定义贝塞尔曲线里面的终点即可

            基本格式为：T x y (or t dx dy)

            <path d="M10 80 Q 95 10 180 80 T 265 80" stroke="black" fill="#ccc"/>

          简单来说，C/S，Q/T 是两两搭配一起使用的

        • 弧线
          • A/a：用来画弧线(Arcs)，起始点是从上一个命令的结束点位置开始计算的

            基准格式：A rx ry x-axis-rotation large-arc-flag sweep-flag x y (or a rx ry x-axis-rotation large-arc-flag sweep-flag dx dy)

            • rx,ry: 代表的就是长轴短轴
            • x,y: 代表的是弧长的结束点。开始点就是上一个命令的终点。
            • x-axis-rotation: x 轴的旋转角度。顺时针为正
            • large-arc-flag[0,1]: 表示取大弧还是小弧。因为两点之间的弧长有两部分。
            • sweep-flag[0,1]: 取顺时针的弧，还是逆时针的弧长。参考点是以起始点开始的。
      • 每种标识符有两种书写方式
        • 大写: 参照的是绝对坐标，即，SVG 的右上角
        • 小写: 参照的相对坐标，即，前一个点的坐标。
    2. 分组：可以理解为 PS 中的图层，分组标签就是 g，使用 g 标签包裹的所有子元素都认同为一组。

      g 标签除了分组，还有另外一个很重要的功能–动画

  7. SVG 复用
    1. defs：预定义一个元素使其能够在SVG图像中重复使用 
      <svg xmlns="http://www.w3.org/2000/svg">
  <defs>
      <g id="shape">
          <rect x="50" y="50" width="50" height="50" />
          <circle cx="50" cy="50" r="50" />
      </g>
  </defs>
</svg>
      • defs使用g的方式来复用图形。
    2. symbol：定义可重复使用的符号 
      <svg xmlns="http://www.w3.org/2000/svg">
	<symbol id="shape2">
		<circle cx="25" cy="25" r="25" style="fill:#bf55ec;"/>
	</symbol>
</svg>
      • symbol元素需要一个ID号，以便以被use元素引用。
      • 一个symbol元素可以有preserveAspectRatio和viewBox属性，而g元素不能拥有这些属性。
    3. use：在SVG图像中多次重用一个预定义的SVG图形，包括g元素和symbol元素 
      <svg xmlns="http://www.w3.org/2000/svg">
  <use xlink:href="#shape" x="50" y="50" />
  <use xlink:href="#shape" x="200" y="50" />
  <use xlink:href="#shape2" x="350" y="50" style="fill:#bf55ec;" />
</svg>
      • 嵌入在defs或symbol元素中的图形是不会被直接显示的，只能使用use元素来引用它。
      • 要引用g元素，必须在g元素上设置一个ID，通过ID来引用它。use元素通过xlink:href属性来引入g元素。
      • 可以在use元素中使用style属性来为复用的图形设置它的样式。
      • 可以引用SVG图像中的任何元素，只要这个元素有一个唯一的ID号： 
        <svg width="500" height="110">
    <g id="shape2">
        <rect x="0" y="0" width="50" height="50" />
    </g>
    <use xlink:href="#shape2" x="200" y="50" />
</svg>

与传统图片格式对比
  1. 是一种可缩放矢量图形

    SVG提供的功能集涵盖了嵌套转换、裁剪路径、Alpha通道、滤镜效果等能力，它还具备了传统图片没有的矢量功能，在任何高清设备都很高清。

  2. 可读性好，有利于SEO与无障碍

    由于SVG采用的是XML语法，图形的里面的文本内容可以直接被浏览器，搜索引擎SEO和无障碍读屏软件读取，设置title与desc标签即可。

    <svg xmlns="http://www.w3.org/2000/svg">
	<title></title>
	<desc></desc>
</svg>
与图标字体(icon font)对比
  1. 清晰度：渲染方式不同

    icon font采用的是字体渲染，浏览器将其视为文字进行抗锯齿优化，有时得到的效果并没有想象中那么锐利，尤其是在不同系统下对文字进行抗锯齿的算法不同，可能会导致显示效果不同。

    SVG使用的是图形渲染，不依赖于分辨率, 因此图形不会因为放大而显示出明显的锯齿边缘。

  2. Icon Font 作为一种字体，Icon 显示的大小和位置可能要受到font-size、line-height、word-spacing等等 CSS 属性的影响。 Icon 所在容器的 CSS 样式可能对 Icon 的位置产生影响，调整起来很不方便。
  3. 使用上存在不便。首先，加载一个包含数百图标的 Icon Font，却只使用其中几个图标，非常浪费加载时间。 自己制作 Icon Font 以及把多个 Icon Font 中用到的图标整合成一个 Font 也非常不方便。
  4. 为了实现最大程度的浏览器支持，可能要提供至少四种不同类型的字体文件。包括TTF、WOFF、EOT 以及一个使用 SVG 格式定义的字体。
  5. 某些浏览器下font-face有跨域问题
  6. SVG可以做到多元素支持，支持彩色图标

    对于图标的每个部分可以有不同的颜色：用不同的颜色填充每个path，也就是通过设置fill属性和stroke属性来控制填充颜色和边线的颜色。

  7. Inline SVG

    Inline SVG，就是直接把 SVG 写入 HTML 中，作为 HTML 文档的一部分，不需要单独请求。临时需要修改某个图标的形状也比较方便
